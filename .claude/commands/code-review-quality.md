# Code Review — Quality

Perform a code review on all modified files. Focus on:

1. **Clean code**: Readability, maintainability, proper naming conventions, single responsibility principle, appropriate function/class sizes
2. **DRY (Don't Repeat Yourself)**: Identify code duplication and suggest refactoring opportunities
3. **Design pattern usage**: Evaluate whether appropriate design patterns are used where beneficial, and suggest improvements where patterns could enhance code quality
4. **Security concerns**: Identify potential security vulnerabilities, input validation issues, authentication/authorization problems, injection risks, and other security best practices
5. **Scalability**: Evaluate that the designed systems will scale well under heavy usage and future scenarios

Review all modified files in the current git status and provide actionable feedback with specific file locations and line numbers where applicable. Be concise and prioritize critical issues.

ONLY provide feedback on things that need changing.
DO NOT provide any positive feedback.
